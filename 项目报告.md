# 语音信号分析系统 — 课程项目报告

## 一、项目要求

本项目设计并实现一个完整的语音信号分析系统，具体要求如下：

1. **音频输入**：支持文件上传、麦克风实时录音、批量文件夹扫描三种方式，兼容 WAV、MP3、FLAC、M4A、OGG 等主流音频格式。
2. **时频域特征提取**：对输入音频提取时域特征（RMS 能量、过零率、波峰因子等）和频域特征（频谱质心、频谱带宽、频谱平坦度、频谱滚降、频谱熵等），并以可视化图表（时域波形图、频谱图、梅尔频谱图）展示。
3. **说话人声纹识别**：基于深度学习声纹模型，实现说话人注册与识别功能，输出识别结果及与声纹库中每位说话人的余弦相似度。
4. **语音识别 (ASR)**：对输入音频进行中文语音转文字。
5. **防伪检测**：识别输入音频是否为 TTS（文本转语音）合成语音，防止声纹系统被合成语音攻击。
6. **JSON 结构化输出**：所有分析结果以 JSON 格式输出并支持下载，便于后续处理与集成。
7. **批量处理**：支持对 `input/` 文件夹内所有音频进行批量分析，生成汇总表格与 JSON 文件。

## 二、系统设计过程及方案

### 1. 项目调研方案

#### 方案一：基于 ECAPA-TDNN 的说话人识别

- **来源**：SpeechBrain 开源语音工具包（[speechbrain/speechbrain](https://github.com/speechbrain/speechbrain)），ECAPA-TDNN 论文 *"ECAPA-TDNN: Emphasized Channel Attention, Propagation and Aggregation in TDNN Based Speaker Verification" (Desplanques et al., Interspeech 2020)*。
- **原理**：ECAPA-TDNN 是当前说话人验证领域 SOTA 架构之一。在传统 TDNN（时延神经网络）基础上引入 SE-Res2Block 通道注意力机制和多层特征聚合，将可变长度语音映射为固定 192 维嵌入向量。系统采用 SpeechBrain 在 VoxCeleb 数据集上预训练的 `spkrec-ecapa-voxceleb` 模型，通过余弦相似度比较嵌入向量实现说话人识别。
- **优势**：预训练模型精度高（VoxCeleb1-O EER < 1%），推理速度快，支持 CPU 运行，适合本地部署。

#### 方案二：基于 Paraformer 的非自回归语音识别

- **来源**：阿里巴巴达摩院 FunASR 框架（[modelscope/FunASR](https://github.com/modelscope/FunASR)），Paraformer 论文 *"Paraformer: Fast and Accurate Parallel Transformer for Non-autoregressive End-to-End Speech Recognition" (Gao et al., 2022)*。
- **原理**：Paraformer 是非自回归端到端语音识别模型，摒弃了传统自回归模型的逐帧解码方式，一次性并行输出完整文本序列。采用 CTC + 预测器 + 采样器的并行架构，推理速度比 Whisper 快 10 倍以上，特别适合中文普通话识别。
- **优势**：中文识别准确率高，内存占用小，CPU 推理友好，且支持从 ModelScope 自动下载模型。

#### 方案三：基于信号底噪分析的 TTS 检测

- **来源**：ASVspoof（自动说话人验证防伪挑战赛）相关研究及音频取证领域通用实践。参考文献：*"ASVspoof 2021: accelerating progress in spoofed and deepfake speech detection" (Yamagishi et al., 2021)*。
- **原理**：TTS 合成语音的音频在生成过程中不经过麦克风采集→模数转换→压缩编码等物理链路，因此其静音段呈现"死寂"特征（RMS 能量极低且异常恒定），与真实语音的麦克风底噪（有自然波动的环境噪声+电路热噪声）存在显著差异。系统基于此物理差异构建了多维启发式评分模型。
- **优势**：不依赖深度学习模型，无需额外下载权重文件，推理零延迟，CPU 友好。

### 2. 核心方法的原理

#### 2.1 时域分析方法

时域分析直接在原始波形采样点序列 y[n] 上进行，反映信号幅度随时间的变化规律。

**(1) 均方根能量 (RMS Energy)**

$$RMS = \sqrt{\frac{1}{N}\sum_{i=1}^{N} y_i^2}$$

RMS 反映信号的总体响度水平。通过对音频进行分帧（帧长 1024 点，帧移 256 点），计算每帧 RMS 后取均值和标准差，可获得整体响度与音量波动信息。

**(2) 过零率 (Zero-Crossing Rate, ZCR)**

$$ZCR = \frac{1}{2N}\sum_{i=1}^{N-1} |\text{sgn}(y_{i+1}) - \text{sgn}(y_i)|$$

ZCR 统计信号穿过零轴的频率，与信号的高频成分占比正相关。清辅音（如 /s/、/sh/）的 ZCR 显著高于浊音（如元音），因此 ZCR 可用于区分清浊音。

**(3) 波峰因子 (Crest Factor)**

$$Crest\ Factor = \frac{\max(|y|)}{RMS}$$

波峰因子衡量信号的冲击性。高波峰因子表示信号中有尖锐的瞬态峰值（如爆破音），低波峰因子表示信号能量分布均匀。

**(4) 峰峰值 (Peak-to-Peak)**

$$P2P = \max(y) - \min(y)$$

表示信号的最大动态摆幅。

#### 2.2 频域分析方法

频域分析通过傅里叶变换将信号从时域映射到频域，揭示频率成分的分布规律。

**(1) 频谱质心 (Spectral Centroid)**

$$SC = \frac{\sum_{k} f_k \cdot |X_k|}{\sum_{k} |X_k|}$$

频谱质心是频谱的"重心"频率，与声音的明亮度/尖锐度相关。元音 /i/ 的质心高于 /a/。

**(2) 频谱带宽 (Spectral Bandwidth)**

$$SB = \sqrt{\frac{\sum_{k} (f_k - SC)^2 \cdot |X_k|}{\sum_{k} |X_k|}}$$

频谱带宽衡量频谱能量围绕质心的分散程度，反映音色的复杂度。

**(3) 频谱平坦度 (Spectral Flatness)**

$$SF = \frac{\sqrt[N]{\prod_{k} |X_k|}}{\frac{1}{N}\sum_{k} |X_k|}$$

几何平均与算术平均的比值，取值 0~1。接近 0 表示频谱有清晰的谐波峰谷结构（调性信号），接近 1 表示频谱平坦（噪声信号）。TTS 合成语音在高频段平坦度通常偏高。

**(4) 频谱滚降 (Spectral Rolloff)**

累积频谱能量达到总能量 85% 时的频率，反映能量集中度。低滚降点表示能量集中在低频（如低语），高滚降点表示高频成分丰富。

**(5) 频谱熵 (Spectral Entropy)**

$$H = -\sum_{k} p_k \cdot \log(p_k), \quad p_k = \frac{|X_k|}{\sum_{k} |X_k|}$$

频谱的信息熵，反映频谱结构的复杂度。真实语音由于声带振动的自然变异，频谱结构更复杂无序。

**(6) 梅尔频谱图 (Mel Spectrogram)**

将线性频率轴通过梅尔刻度映射到模拟人耳感知的非线性频率轴：

$$mel(f) = 2595 \cdot \log_{10}(1 + f/700)$$

通过 80 通道梅尔滤波器组将 STFT 频谱转换为梅尔频谱，再转换为 dB 尺度。梅尔频谱图是语音分析中最重要的可视化工具之一——横轴为时间，纵轴为梅尔频率，颜色深度代表能量。从图中可直接观察共振峰（F1/F2/F3）轨迹，这是区分 TTS 与真实语音的关键视觉依据。

#### 2.3 TTS 检测原理

本系统采用**基于物理链路的底噪分析**进行 TTS 检测。核心原理如下：

真实语音的产生和采集链路为：
$$人声 → 麦克风振膜 → 前置放大器 → ADC模数转换 → 数字音频$$

这一链路中存在两个不可避免的噪声源：(1) 环境背景噪声（空气传导到麦克风），(2) 麦克风和前置放大器的电路热噪声。即使在"安静"环境下录制，ADC 采集到的底噪 RMS 通常也在 $10^{-5} \sim 10^{-3}$ 量级，且底噪存在自然的时变波动（变异系数 $CV > 0.4$）。

TTS 合成语音的生成链路为：
$$文本 → 声学模型 → 神经声码器 → 数字波形$$

全程在数字域完成，不涉及模拟器件。因此 TTS 音频的静音段（词语间停顿、句首句尾）呈现**数字"死寂"**：RMS 能量极低（可能为精确的 $0$ 或 $< 10^{-6}$），且底噪异常恒定（$CV < 0.15$）。

系统通过以下三个维度构建评分模型：

| 维度 | 权重 | 原理 | 真人特征 | TTS 特征 |
|------|:----:|------|----------|----------|
| 底噪稳定性 | 0.50 | 静音段 RMS 的变异系数 | CV > 0.4（自然波动） | CV < 0.15（异常恒定） |
| 高频平坦度波动 | 0.30 | 3kHz 以上频谱平坦度的帧间标准差 | Std > 0.12（谐波自然起伏） | Std < 0.07（均匀平滑） |
| 底噪强度 | 0.20 | 静音段平均 RMS 量级 | $> 5 \times 10^{-4}$（有底噪） | $< 10^{-6}$（死寂） |

最终 TTS 分数为三个维度的加权和，取值范围 0~1（0 = 确定 TTS，1 = 确定真人）。

### 3. 参数选择理由

#### 3.1 音频参数选择

| 参数 | 取值 | 理由 |
|------|------|------|
| 采样率 | 16000 Hz | ECAPA-TDNN 和 Paraformer 均要求 16kHz 输入；语音频带主要信息集中在 8kHz 以下（奈奎斯特频率），16kHz 是语音处理的标准采样率 |
| 单声道 | Mono | 声纹和 ASR 模型均要求单声道输入，立体声的双通道对语音分析无额外信息增益 |
| 时域帧长/帧移 | 1024/256 | 1024 点 @16kHz = 64ms 帧长，兼顾时间分辨率和统计稳定性；256 点帧移 = 16ms，与语音的短时平稳假设一致 |
| 频域 FFT 点数 | 1024 (特征)/4096 (可视化) | 特征提取时 1024 点平衡了频率分辨率和计算效率；频谱可视化时 4096 点提供更精细的频率分辨率以观察谐波细节 |

#### 3.2 声纹识别参数选择

| 参数 | 取值 | 理由 |
|------|------|------|
| 嵌入维度 | 192 | ECAPA-TDNN 标准输出维度，由预训练模型固定 |
| 相似度度量 | 余弦相似度 | 嵌入向量经 L2 归一化后，余弦相似度等于内积，高效且与训练时的度量一致 |
| 相似度阈值 | 0.62 | 在 VoxCeleb 数据集上验证的推荐阈值，平衡误拒率 (FRR) 和误识率 (FAR) |
| 注册音频数 | 3~5 条/人 | 足够计算稳定的质心嵌入，同时保持注册流程简洁 |

#### 3.3 ASR 参数选择

| 参数 | 取值 | 理由 |
|------|------|------|
| 模型 | Paraformer-large | 中文 ASR SOTA，非自回归架构推理快，CPU 友好 |
| 语言 | 中文 (zh) | 课程项目目标语言 |
| 线程数 | ncpu=2 | 限制 CPU 占用，避免与声纹模型竞争资源导致内存溢出 |

#### 3.4 TTS 检测参数选择

| 参数 | 取值 | 理由 |
|------|------|------|
| 静音帧阈值 | 15% 分位数 | 取最安静的 15% 帧作为静音候选，既能捕获真实底噪又避免语音段干扰 |
| 底噪稳定性 CV 映射 | (cv - 0.05) / 0.50 | 基于实测校准：真人 cv ≈ 0.52，TTS cv ≈ 0.09。线性映射使真人得分 ≈ 0.95，TTS 得分 ≈ 0.08 |
| 高频平坦度 Std 映射 | std / 0.12 | 基于实测校准：真人 std ≈ 0.160，TTS std ≈ 0.066 |
| TTS 检测阈值 | 0.40 | 低于此阈值判定为 TTS。基于实测：真人 ≈ 0.87，TTS ≈ 0.34，阈值 0.40 留有安全边际 |

#### 3.5 防伪检测阈值的实测调优过程

初始方案采用基于说话人注册分布对比的 `profile_score`（Z-score 指数衰减），但该方案将"注册时的麦克风特性"错误地当成了"真人声学指纹"，导致同一人换麦克风后分数极低。随后尝试了频谱平坦度/频谱熵/RMS 变异系数等手工特征，以及 ECAPA 分类器熵，均因现代 TTS（CosyVoice、HiFi-GAN 神经声码器）生成的频谱结构与真人几乎无差别而失败。

**关键发现**：在反复测试中发现，只有**静音段底噪稳定性 (noise floor CV)** 和**高频平坦度帧间波动 (hf_flat_std)** 两个维度在真人和 TTS 之间存在系统性差异。最终基于 5 轮实测数据迭代校准，确定了表 3.4 中的映射参数和权重分配。

**主要功能模块说明：**

1. **音频输入模块**：提供三种输入方式（上传文件、麦克风录音、批量扫描 `input/` 文件夹），兼容 WAV/MP3/FLAC/M4A/OGG 格式。
2. **音频预处理**：通过 librosa 将任意格式音频统一重采样为 16kHz 单声道，确保后续模块输入一致性。
3. **时频特征提取**：分时域（8 个特征）和频域（10 个特征）两个维度，并通过 matplotlib 生成时域波形图、FFT 频谱图和梅尔频谱图。
4. **声纹注册模块**：读取 `register_id/` 下按说话人命名的子文件夹，每人 3~5 条音频，提取 ECAPA-TDNN 嵌入后计算质心向量存入声纹库 JSON。
5. **声纹识别模块**：提取输入音频的 192 维嵌入，与声纹库中所有质心计算余弦相似度，取最高分说话人作为识别结果。
6. **TTS 检测模块**：基于静音段底噪分析的三维启发式评分模型，输出 0~1 的综合 TTS 分数。
7. **ASR 识别模块**：Paraformer-large 模型进行中文语音转文字。
8. **结果合成与输出**：汇总所有分析结果，以 JSON 和表格形式呈现，支持下载。

### 2. 调试步骤

**步骤 1：环境搭建与模型加载**

安装依赖库（streamlit, torch, speechbrain, funasr, librosa, soundfile），下载 ECAPA-TDNN 预训练权重和 Paraformer 模型，验证模型能正常加载到 CPU。

**步骤 2：声纹注册与识别基线测试**

在 `register_id/` 下为 2~3 位说话人各放入 3~5 条音频，运行注册功能。用其中一人的另一条音频测试识别功能：
- 验证相似度 > 0.62（阈值），且最高相似度对应正确说话人。
- 调整阈值在 0.40~0.95 范围滑动，观察对识别结果的影响。

**步骤 3：TTS 检测校准**

分别录制一段真人语音和生成一段 TTS 合成语音（使用 CosyVoice 等现代 TTS 系统），测试 TTS 检测模块：
- 第 1 轮：验证静音段底噪 RMS 是否存在显著差异。
- 第 2 轮：发现底噪 CV（变异系数）差异更显著（真人 0.52 vs TTS 0.09），调整权重优先使用 CV。
- 第 3 轮：加入高频平坦度标准差（真人 0.16 vs TTS 0.07）作为辅助维度。
- 第 4 轮：去除无区分力的特征（频谱通量、倒谱峰、ECAPA 分类器熵）。
- 第 5 轮：微调映射函数参数，使真人得分 > 0.8、TTS 得分 < 0.4，差距最大化。

**步骤 4：ASR 识别验证**

用标准中文普通话测试音频（如"今天天气很好"），验证 Paraformer 输出文本的准确性。

**步骤 5：端到端联调与 JSON 输出验证**

完成一次完整的"输入→分析→输出"流程，检查 JSON 输出的字段完整性、精度正确性，以及下载功能。

**步骤 6：UI 优化**

调整页面布局（wide 模式）、标签页结构、图表尺寸、帮助文本（tooltip），确保操作流程清晰流畅。

## 四、结果分析

### 1. 判定功能完成质量的依据

系统的核心功能可通过以下指标体系评估：

**(1) 声纹识别**

| 指标 | 预期 | 实测 |
|------|------|------|
| 同说话人相似度 | ≥ 0.70 | 0.80~0.95（取决于录音质量） |
| 不同说话人相似度 | ≤ 0.50 | 0.20~0.40 |
| 分离度（同人 min - 异人 max） | > 0.20 | > 0.30 |

结论：系统能有效区分已注册的说话人。

**(2) TTS 检测**

**(4) ASR 识别**

Paraformer 模型对标准普通话的识别准确率可达 95% 以上（在安静环境下），满足中文语音转文字需求。

### 2. 系统性能的定性分析

**(1) 优势**

- **架构清晰**：模块化设计，声纹识别、TTS 检测、ASR 三大核心功能独立、可替换。
- **参数可调**：所有阈值通过 UI 滑块实时调整，便于根据实际场景优化。
- **可视化完善**：时域波形、FFT 频谱、梅尔频谱图、TTS 检测子分数明细均有展示，透明度高。
- **CPU 可运行**：所有模型均支持 CPU 推理，无需 GPU，适合本地部署。
- **JSON 输出标准化**：分析结果以结构化 JSON 输出，便于接入下游系统或数据库。

**(2) 局限性**

- **TTS 检测的泛化性**：底噪分析法依赖于音频中存在静音段。如果 TTS 音频被人工加入模拟噪声（如加性高斯白噪声），底噪特征可能被混淆。对于专为绕过此类检测而优化的 TTS 系统，准确率会下降。
- **声纹识别的环境敏感度**：ECAPA-TDNN 对麦克风类型、房间混响、背景噪声敏感。注册和测试使用不同设备时，相似度会下降。
- **单语言 ASR**：Paraformer 仅支持中文，不适用于多语言混合语音。
- **未利用 TSE 模型**：项目中存在目标说话人提取 (TSE) 模型文件，但未在系统流程中集成。

## 五、项目总结

### 1. 设计过程中遇到的主要困难和解决方案

**困难 1：TTS 检测的多次迭代失败**

最初的方案是基于说话人注册数据的 `profile_score`（Z-score），但该方案将"录音设备特性"误判为"说话人特征"，导致同一人换设备后分数剧降。随后尝试了频谱平坦度均值、频谱熵、RMS 变异系数、HPSS 谐波噪声比、调制谱能量、ECAPA 分类器熵等多个方案，均因现代 TTS 的声学质量极高而无法区分。

**解决方案**：回到音频物理链路层面思考——TTS 是纯数字生成，没有经过麦克风和 ADC。最终定位到**静音段底噪稳定性**和**高频平坦度帧间波动**两个核心区分维度，经过 5 轮实测迭代校准了 3 维加权评分模型。

**困难 2：多模型同时加载导致内存溢出**

ECAPA-TDNN + Whisper + Paraformer 三个大模型同时驻留内存时容易 OOM。

**解决方案**：(1) 使用 `torch.set_num_threads(2)` 限制 PyTorch 线程数；(2) 移除 Whisper 模型，仅保留中文效果更好的 Paraformer。

**困难 3：faster-whisper 与 Paraformer 模型冲突**

faster-whisper 在某些 Windows 环境下与 FunASR 的依赖版本不兼容，导致 `DLL load failed`。

**解决方案**：移除 Whisper/融合输出路径，从三引擎融合简化为 Paraformer 单引擎，既解决了兼容性问题，又减少了内存占用和推理时间。

### 3. 项目的结果小结、未完成或待完善部分的展望

**已完成**：系统实现了音频输入→时频特征提取→声纹识别→TTS 检测→ASR 识别→JSON 输出的完整链路，Web 界面操作流畅，所有参数可调节。

**待完善**：

1. **TTS 检测的深度学习化**：当前启发式模型依赖静音段，未来可引入 ASVspoof 挑战赛的预训练模型（如 Nes2Net、AASIST）作为可选的深度学习后端，提升对不同 TTS 系统的泛化能力。
2. **目标说话人提取 (TSE) 集成**：项目中已有 BSRNN+ECAPA 的 TSE 模型文件，可在多人重叠语音场景中提取目标说话人后单独分析。
3. **多语言 ASR 支持**：Paraformer 仅支持中文，可增加 SenseVoice 等多语言模型以支持中英混合场景。
4. **实时流式处理**：当前为离线文件处理模式，可扩展为实时音频流处理（如 WebSocket 推送麦克风数据流）。
5. **声纹库管理界面优化**：增加说话人删除、重命名、合并等管理操作。

### 4. 对课程项目的意见和建议

1. 课程可增加对现代 TTS/语音合成技术的介绍，让学生了解当前防伪检测面临的挑战。
2. 建议在评分标准中增加对系统实际测试数据（如真人和 TTS 的区分度、声纹识别准确率）的量化要求，鼓励学生在真实场景下验证系统。
3. 可提供一组标准化的测试音频集（包含真人和 TTS 样本），使各组的评估结果具有可比性。
